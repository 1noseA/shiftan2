---
name: create-pr
description: |
  GitHubにPull Requestを作成するスキル。コミット差分からPR本文（日本語）を自動生成し、
  ユーザーに内容を確認させてから `gh pr create` を実行する。
  以下のような発言でこのスキルを使う：
  "PRを作って"、"PR作成"、"PR出して"、"プルリクを作って"、"GitHubにPRを出したい"、
  "pull requestを作成して"、"プルリクエストを出して"、"レビュー依頼したい"
---

## 手順

### 1. コンテキスト収集

以下を並列実行してリポジトリ状態を把握する：

```bash
git branch --show-current
git log main..HEAD --oneline          # main がなければ master を試す
git diff main...HEAD --stat
git status --short
```

- コミットが0件なら「プッシュ済みのコミットがありません」と伝えて終了する
- 現在のブランチが `main` / `master` なら警告して確認する

**関連Issueの特定：**

会話の流れや引数からIssue番号が読み取れる場合はそれを使う。
不明な場合は `gh issue list --limit 10` で一覧を取得し、PR内容と照合して候補を提示する。
それでも特定できなければ「関連Issueはありますか？（番号を教えてください）」と確認する。

### 2. PRドラフト作成

収集した差分・コミットログをもとに **日本語** で下書きする。

**タイトル（70文字以内）**
スプリント名・機能名など「何をしたか」を端的に。
例：`Sprint 1: スタッフ管理機能の実装`、`fix: シフト生成のタイムアウト処理を修正`

**本文テンプレート：**

```
## 概要
（このPRで何をしたか・なぜしたかを2〜3文で）

## 変更内容
- （変更点を箇条書き。ファイル羅列ではなく意図・効果を書く）

## 動作確認
- （実際に確認した内容を過去形で書く。例：「スタッフ招待フォームから招待メールが送信されることを確認した」）

## レビューしてほしいこと
- （設計判断・実装方針など、レビュアーに特に見てほしい点）

## 関連Issue
- Closes #<Issue番号>（関連Issueがない場合は「なし」と書く）

🤖 Generated with [Claude Code](https://claude.ai/claude-code)
```

### 3. ユーザーへの確認

ドラフトを以下の形式で提示し、承認・修正・キャンセルを求める。
`gh pr create` はこの確認が取れるまで **絶対に実行しない**。

---
**【PR確認】**

**タイトル：**
```
<タイトル>
```

**本文：**
```
<本文>
```

このPRを作成しますか？（「はい」で作成 / 内容を指示すれば修正 / 「キャンセル」で中止）

---

修正を求められたら内容を反映して再度提示する。承認されたら次のステップへ。

### 4. PR作成

```bash
gh pr create --title "<タイトル>" --body "$(cat <<'EOF'
<本文>
EOF
)"
```

作成後、返ってきた **PR の URL** をユーザーに伝えて終了。

## エラー対応

| 状況 | 対応 |
|---|---|
| `gh: command not found` | `brew install gh` を案内 |
| 未認証エラー | `gh auth login` を案内 |
| リモートブランチ未プッシュ | `git push -u origin <branch>` を促してから再実行 |
